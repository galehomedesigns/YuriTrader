# Opening-Power — single-day A/B backtest, the 60th trading day back (2026-03-26)

**Question:** on one historical day, how much would $1,000 have made under the strategy
**as configured** vs. the **"small change"** (turn the coil filter off, enter on the
9:30 breakout) — and is the change worth pursuing?

**Day chosen:** 2026-03-26 (Thu) = the 60th NYSE trading session before today
(2026-06-23). Verified against the live session calendar (130 daily bars pulled; today
and 2026-06-22 both present, so the count is exact).

**Data:** IBKR native 2-min RTH bars (the backtest vendor), pulled fresh per symbol with
≥200 prior bars so the SMA200 gate is real. Universe = the 72 IBKR-cached tech names (the
same set the validated backtests use). All 72 had enough history.

---

## Methodology (identical for both scenarios except the selection rule)

Pulled from the live config in `.env` + code defaults — **not** guessed:

| Rule | Value | Source |
|---|---|---|
| TIGHT gate | **ATR-normalized**, sep/ATR ≤ 1.0, ATR len 14 | `.env` `OPENING_TIGHT_MODE=atr` |
| Arm window | re-classify each completed 2-min bar **9:30→9:42** | `OPENING_ARM_WINDOW_MIN=15` |
| Entry | buy-STOP at signal-bar high + $0.01 | `classifier.entry_level_long` |
| Stop | signal-bar low − $0.01 | `classifier.stop_level_long` |
| Exit | protective stop, else close of the +30-min cutoff bar (10:00 ET) | `OPENING_SESSION_CUTOFF_MIN=30`, EOD=flatten |
| Sizing | equal **$200 dollar slots** ($1,000 ÷ 5), max 5 names | `_stage_entries` |
| Risk cap | skip if (entry−stop)/entry > 3% | `OPENING_MAX_RISK_PCT=3.0` |
| Min range | skip if entry−stop < $0.05 | `OPENING_MIN_BAR_RANGE` |

> Sizing note: the live engine enters a **half-slot** first ($100) and completes to the
> full slot via the G9 pull-back add. This run prices **full $200 slots** (the add
> assumed to fill) — an upper bound on position size. Halve the dollar figures for the
> no-add case. Commissions (~$1/order) are **not** included.

- **Scenario A — AS-BUILT:** coil gate ON. Rolling-arm across 9:30–9:42; the first bar
  that classifies `MATCH_LONG` arms the name; take the first 5 by arm time (after the
  risk cap).
- **Scenario B — SMALL CHANGE:** coil gate OFF. Rank the universe by overnight gap-up
  (today's 9:30 open vs. prior session close); take the top-5 gappers; enter on the **9:30**
  bar's breakout. (Gap is the strategy's one independently-validated edge, so it's the
  fair gate-free selector.)

## Mistakes I made earlier in this thread (so they're on the record)

Each of these pushed the numbers **against** the opportunity, which is why the figures
kept getting revised. Listed so the pattern is visible and fixable.

1. **Only classified the 9:30 bar — omitted the rolling-arm window.** The strategy
   re-checks every new 2-min bar to ~9:42 (`OPENING_ARM_WINDOW_MIN=15`). I therefore
   reported **"$0, nothing filled"** for 2026-06-23 when the strategy actually arms ~6
   names. This is the "you weren't including the 2-minute criteria" point — correct, and
   it was the biggest error.
2. **Didn't load `.env`, so I used the wrong TIGHT gate.** The code default is a flat
   0.25% band; the **live** gate is ATR-normalized (`OPENING_TIGHT_MODE=atr`). Every
   TIGHT/WIDE verdict in my first table was computed on the wrong test.
3. **Reported P&L per-share (1 share each) instead of the strategy's equal-dollar
   slots.** Per-share over-weights expensive stocks; it made the rejected set look
   net-negative (−$1.66/share) when equal-dollar sizing is net-**positive** (+$8). A
   sizing-convention error that mis-framed the whole question.
4. **Editorialized conclusions the data didn't support** — "gate net-correct" and
   especially **"no hidden winners," which was simply false** (the rejected set held
   QBTS +5%, INFQ +3.6%, EPC +2.5%). I retract those phrasings.

**Root cause:** I kept hand-rolling a *simplified* model (first bar only, wrong gate,
wrong sizing) and narrating conclusions, instead of faithfully reproducing the live
mechanics from config and letting the numbers speak. The fix is the methodology table
above + running the canonical tooling rather than re-deriving it each time.

## Caveats on this result
- **One day.** Both scenarios green; the +$3 edge is one stock (MARA). Not evidence.
- **Vendor:** IBKR 2-min bars, not the TradingView feed the live engine trades on; names
  on the TIGHT knife-edge can flip between vendors.
- **Upper-bound sizing:** full $200 slots (G9 add assumed filled); no commissions.
- The coil gate was A/B-validated as *net-positive over ~60 days* in prior work — so a
  single day where gate-off wins by $3 is well within noise. The honest next step is the
  full multi-day sweep of exactly these two scenarios.
